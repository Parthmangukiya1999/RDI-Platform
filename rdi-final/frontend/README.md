## рџ”Њ API Integration

All API calls are centralized in `src/services/api.js`.

### Services Available:
- **authService** - Login, Register, OAuth
- **userService** - Profile management
- **teamService** - Team CRUD operations
- **gamesService** - Game data
- **playerService** - Player search
- **searchService** - Global search
- **careerService** - Job applications
- **communicationService** - Messaging

### Example Usage:
```javascript
import { authService, teamService } from './services/api';

// Login
const response = await authService.login(email, password);

// Get all teams
const teams = await teamService.getAll({ game: 'Counter-Strike 2' });

// Create team
const newTeam = await teamService.create(teamData);
```

### Backend Requirements:
The frontend expects a REST API on `http://localhost:5000` with 35+ endpoints.

**See**: [FRONTEND_HANDOFF_TO_BACKEND.md](../FRONTEND_HANDOFF_TO_BACKEND.md) for complete API documentation.

## рџ”’ Authentication Flow

1. User registers/logs in
2. Backend returns JWT token
3. Token stored in `localStorage`
4. All API requests include token in headers
5. On 401 response, user logged out automatically

```javascript
// Token included automatically in all requests
headers: {
  'Authorization': `Bearer ${localStorage.getItem('token')}`
}
```

## рџљЁ Common Issues

### Backend Not Running
```
Error: Network Error / Failed to fetch
```
**Solution**: Ensure backend is running on port 5000.

### CORS Error
```
Access to fetch blocked by CORS policy
```
**Solution**: Backend must allow `http://localhost:5173` in CORS config.

### Token Expired
```
Session expired. Please login again.
```
**Solution**: Normal behavior. Token expires after 7 days.
